Introduction to Programing

HTML and the "Web"
Intro

The World Wide Web is made up of HTML or Hypertext Markup Language documents. Approximately 30 billion documents are "webbed" by hyperlinks. There are three major pieces that make up the web: a user with a computer and a browser that "reads" webpages,the internet (uses HTTP to make and send requests to servers), and servers, computers optimized to host the documents that make up the web.

HTML - Hypertext Markup Language

HTML is made up of four main components: text content (what you can see), markup (creates the "look" of the page), references to other documents (e.g. images and videos), and links to other pages.

Markup - Tags and Elements

Elements are made up of tags (both opening tags, and closing tags) with content between the tags. These tags instruct the computer to change the content between them in a particular way (e.g. making text bold or italics).

One type of tag, known as an attribute, can be used to create a link in the text (among other functions). For example: Practice HTML Here! Tags can also be used to insert images onto a page. Keep in mind that tags create elements that are either inline or block. Inline elements are on the line with text while block elements create an "invisible box" around the content.

Computers are Stupid

Computers read instructions literally. For example: if you use a bold opening tag and failed to use a closing tag, the computer would interpret that as you asking for everything after the opening tag to be in bold text.

Some General Concepts to Remember

1 - Webpages are text documents written in HTML and "read" by browsers.

2 - Coding is instructions written in computer language.

3 - Computers are stupid!

4 - Programmers can't remember everything. Don't be afraid to look things up!

5 - Basic HTML vocabulary: tag, element, attribute.

Telling Computers What to Do

We uses programming "languages" to give computers instructions. We use code languages rather than natural languages beause of computer languages are much more efficient. Natural languages are also inherently ambiguous.

In this course, we will use Python. Python is a "high level language." This means that Python runs through an interpreter (also called Python) rather than directly on the computer.

Language Structure: Backus-Naur Form

Backus-Naur Form is a way of discribing languages by generalizing "grammer structure." An initial form is described and expanded with non-terminals until a terminal is reached.

We can demonstrate this using english grammar rules. We describe an intital form of a sentence as subject-verb-object. Subjects and objects are given a non-terminal of nouns and those nouns are given terminal options of specific words of noun form.

Variables, Strings, Indexing, Sub-Sequences, and Find

"Variables" are names that are assigned to expressions in the form: name = expression. The name must start with an integer or an underscore. The equal sign is not the same as equivalency in math. Rather, in coding, the equal signs means assignment.

"Strings" are denoted by single or double quotes and can be letters, numbers or the "empty string." Adding two strings together is called "concatenation."

It is possible to "index" strings, or find the value of a string at a given position. This is achieved by printing the variable followed by brackets "[ ]" with a number (beginning at "0" for the first position).

Similarly, a "sub-sequence" in a string can be identified by "[a:b]" where "a" is the starting position and "b" is the character one position before the value of "b."

The position of a "search string" can be found within a "target string" by using ".find" (This will return the first occourance or -1 if the search string isn't found. To find further occurances assign a paramter.).

NOTE: Keep in mind that 2 + 2 is NOT the same as "2" + "2"

Functions

Functions have two parts: "def" and "return". "Def" is made up of the function name followed by operands (aka inputs) followed by a colon. The "return" portion of the function is indented below "def" and specifies what is done to create the function's output.

NOTE: if the return is not specified then the function will return a value of "none"

Once a function is definied it can be used repeatedly eliminating repetition!

Debugging

Before beginning, make sure that the problem is accurately defined. Problems are defined by a set of possible inputs and their relationship to desired outputs.

Don't jump straight to coding and don't optimize prematuraly. Use a systematic approach solving small parts of the larger problem one by one and synthesizing into a complete solution.

Structured Data: Lists and For Loops

A string is denoted by quotes (single, double or triple) while lists are a sequence of anything (strings or another element including other lists) denoted by brackets. You can mutate a list by reassigning one of the members of the list a different value. Three list operations: "Append" (adds elements to the end of a list); adding of two lists; "len" returns the number of elements in a list.

How to Solve Problems

1- Understand the problem and split into different parts. 2- Define inputs and outputs. 3- Write "pseudo code". 4- Look for the simplest solution. 5- Do not optimize prematurely.
